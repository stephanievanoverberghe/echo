# 🎨 Étape 09 — Poser la base CSS globale

---

## 🧭 Objectif

Définir les fondations CSS du projet avant de styliser les pages et les composants en détail.

Cette étape permet de :

- créer une base propre et cohérente
- éviter les styles dispersés
- poser l’identité visuelle générale du site
- préparer un code CSS lisible et maintenable

---

## 🧠 Intention

La base CSS doit installer immédiatement l’atmosphère du projet.

Même sans styles détaillés, le site doit déjà donner une impression de :

- profondeur
- silence
- obscurité maîtrisée
- élégance minimale
- respiration visuelle

---

## 🧱 Fondations techniques

La feuille de style principale doit d’abord poser des règles globales simples et saines.

### À prévoir

- une base de reset légère
- une règle de dimensionnement cohérente
- la suppression des marges par défaut inutiles
- des images fluides
- des liens propres
- une base stable pour tous les éléments

---

## 🎨 Variables CSS globales

Le projet doit définir des variables globales afin de centraliser les choix visuels.

### Catégories de variables à prévoir

#### Couleurs

- couleur de fond principale
- couleur de fond secondaire
- couleur de surface
- couleur de texte principal
- couleur de texte secondaire
- couleur d’accent
- couleur de bordure subtile

#### Typographie

- police principale
- police secondaire éventuelle
- tailles de titres
- tailles de texte courant
- hauteurs de ligne

#### Espacements

- petits espacements
- espacements moyens
- grands espacements
- espacements verticaux entre sections

#### Formes

- rayons de bordure
- largeur maximale du contenu

#### Effets

- ombres
- halos
- transitions
- flous éventuels

---

## 🌌 Ambiance générale à installer

Le CSS global doit faire apparaître :

- un fond sombre mais nuancé
- un texte lisible et doux
- des contrastes subtils
- une sensation d’espace
- une lecture confortable

Il faut éviter :

- les contrastes violents
- le noir pur
- le blanc pur
- les éléments trop rigides visuellement

---

## 🖋️ Styles de base des éléments principaux

Des styles de base doivent être définis pour les éléments HTML les plus importants.

### Éléments concernés

- body
- headings
- paragraphes
- liens
- images
- listes
- sections
- boutons
- champs de formulaire

L’objectif est de :

- harmoniser leur apparence
- garder une cohérence visuelle
- éviter les styles par défaut trop bruts

---

## 📐 Layout global

Le projet doit définir une structure de mise en page récurrente.

### À prévoir

- une largeur maximale pour le contenu
- un centrage cohérent
- des marges horizontales adaptées
- des espacements verticaux réguliers
- une respiration entre les sections

Le layout doit rester :

- fluide
- simple
- lisible
- facile à réutiliser sur toutes les pages

---

## 📱 Base responsive

Le CSS global doit être pensé en mobile first.

### Cela implique

- des largeurs souples
- des images adaptatives
- des blocs qui peuvent s’empiler naturellement
- une lecture confortable sur petit écran

Le responsive ne doit pas être un ajout tardif.  
Il doit être anticipé dès la base.

---

## 🧩 Préparation des composants

Même si les composants ne sont pas encore stylisés en détail, la base CSS doit déjà préparer :

- le rythme visuel
- les zones de contenu
- la typographie
- la cohérence de navigation
- l’équilibre général du site

---

## 🧼 Résultat attendu

À la fin de cette étape :

- la page a déjà une base visuelle propre
- le site possède une identité globale
- le CSS est structuré pour accueillir les composants ensuite
- le projet paraît déjà plus cohérent, même sans design avancé

---

## ✅ Validation de l’étape

L’étape est validée si :

- une base CSS globale est pensée
- les variables principales sont définies
- les styles HTML de base sont prévus
- le layout global est posé
- le responsive est anticipé
- l’ambiance générale est déjà perceptible
