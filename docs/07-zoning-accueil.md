# 🧭 Étape 8 — Zoning de la page d’accueil

---

## 🧭 Objectif

Définir la structure visuelle de la page d’accueil.

Cette étape permet de :

- organiser les éléments à l’écran
- structurer la lecture
- poser les bases du design avant le CSS

---

## 🧠 Principe du zoning

Le zoning consiste à :

- découper la page en blocs
- définir leur position
- organiser le flux de lecture

On ne parle pas encore de design précis, mais de :

- structure
- hiérarchie
- disposition

---

## 🏗️ Structure globale de la page

La page d’accueil est composée de :

1. Header
2. Hero
3. Introduction
4. Navigation vers les pages
5. Section immersive finale
6. Footer

---

## 🧭 Flux de lecture

L’utilisateur doit :

1. voir une ambiance forte (hero)
2. comprendre qu’il est dans un univers particulier
3. lire un premier fragment
4. être invité à explorer
5. continuer naturellement

---

## 🦸 Zone 1 — Header

Position :

- en haut de la page

Contenu :

- nom du projet
- navigation principale

Objectif :

- orienter sans casser l’immersion

---

## 🌌 Zone 2 — Hero

Position :

- premier écran visible

Contenu :

- phrase forte ou mystérieuse
- ambiance visuelle

Objectif :

- capter immédiatement l’attention
- poser l’atmosphère

---

## 📖 Zone 3 — Introduction

Position :

- juste après le hero

Contenu :

- court texte narratif

Objectif :

- donner un premier sens
- sans tout expliquer

---

## 🧭 Zone 4 — Accès aux pages

Position :

- après l’introduction

Contenu :

- liens ou blocs vers :
    - Journal
    - Artefacts
    - Carte
    - Trace

Objectif :

- encourager l’exploration
- rendre la navigation évidente

---

## 🌫️ Zone 5 — Section immersive

Position :

- bas de page

Contenu :

- texte ou élément visuel
- ambiance forte

Objectif :

- prolonger l’expérience
- laisser une impression

---

## 🧾 Zone 6 — Footer

Position :

- tout en bas

Contenu :

- texte simple
- signature visuelle

Objectif :

- clôturer sans casser l’ambiance

---

## 📏 Règles de disposition

- beaucoup d’espace entre les sections
- lecture verticale fluide
- pas de surcharge
- hiérarchie claire

---

## 🧠 Hiérarchie visuelle

Priorité :

1. Hero
2. Texte d’introduction
3. Accès aux pages
4. Élément final

Chaque zone doit être identifiable immédiatement.

---

## 🧪 Comportement attendu

- le scroll doit être naturel
- les sections doivent apparaître progressivement
- l’utilisateur ne doit jamais être perdu

---

## ✏️ Représentation mentale (wireframe simplifié)

Header

↓

Hero (plein écran)

↓

Introduction (texte centré)

↓

Navigation (blocs ou liens)

↓

Section immersive

↓

Footer

---

## ✅ Validation de l’étape

L’étape est validée si :

- les zones sont définies
- leur ordre est logique
- le flux de lecture est clair
- la hiérarchie visuelle est posée
- la page peut être imaginée sans design détaillé
