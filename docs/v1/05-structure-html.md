# 🏗️ Étape 5 — Définir la structure HTML sémantique

---

## 🧭 Objectif

Définir la structure HTML de chaque page avant de commencer le développement.

Cette étape sert à :

- organiser le contenu proprement
- utiliser les bonnes balises
- préparer un code clair, logique et maintenable

---

## 🧠 Principes à respecter

La structure HTML doit être :

- sémantique
- lisible
- cohérente
- hiérarchisée

Chaque page doit avoir :

- une structure globale claire
- un ordre logique
- une hiérarchie de titres correcte

---

## 🏠 Structure générale commune à toutes les pages

Chaque page du site doit contenir :

### 1. Header

Rôle :

- contenir l’identité du site
- afficher la navigation principale

### 2. Main

Rôle :

- contenir le contenu principal unique de la page

### 3. Footer

Rôle :

- clôturer la page
- afficher une signature, un texte ou une navigation secondaire

---

## 🧱 Structure sémantique attendue

Dans l’ensemble du site, tu vas utiliser les grandes zones HTML suivantes :

- en-tête de page
- navigation principale
- contenu principal
- sections de contenu
- articles pour les fragments du journal
- figures pour les éléments visuels
- formulaire pour la page Trace
- pied de page

---

## 🏠 Structure sémantique de la page Accueil

### Zones principales

- Header
- Main
- Footer

### Dans le Main

- une section Hero
- une section Introduction
- une section Navigation vers les autres pages
- une section de fin ou transition

---

## 📜 Structure sémantique de la page Journal

### Zones principales

- Header
- Main
- Footer

### Dans le Main

- une section d’introduction
- une section contenant la liste des fragments
- chaque fragment doit être pensé comme un contenu autonome

Les fragments peuvent être structurés comme :

- un article
- avec un titre
- un texte
- un éventuel repère temporel ou symbolique

---

## 🧿 Structure sémantique de la page Artefacts

### Zones principales

- Header
- Main
- Footer

### Dans le Main

- une section d’introduction
- une section galerie

Chaque artefact peut être structuré comme :

- un bloc autonome
- avec une image
- un titre
- une description
- une légende éventuelle

---

## 🗺️ Structure sémantique de la page Carte

### Zones principales

- Header
- Main
- Footer

### Dans le Main

- une section d’introduction
- une section contenant les différentes zones

Chaque zone peut être structurée comme :

- un article ou un bloc descriptif
- avec un nom
- une courte description
- un lien éventuel

---

## ✍️ Structure sémantique de la page Trace

### Zones principales

- Header
- Main
- Footer

### Dans le Main

- une section d’introduction
- une section contenant le formulaire

Le formulaire doit contenir :

- un champ pour le nom
- un champ pour le message
- un bouton d’action

---

## 🔠 Hiérarchie des titres

Chaque page doit respecter une hiérarchie logique :

### Niveau principal

- un seul titre principal par page

### Niveaux secondaires

- titres de section
- sous-titres éventuels si nécessaire

L’objectif est de :

- structurer le contenu
- améliorer la lisibilité
- préparer une bonne accessibilité

---

## 🧭 Navigation

La navigation doit être présente sur toutes les pages.

Elle doit permettre :

- d’accéder aux pages principales
- de revenir à l’accueil
- de garder une cohérence dans l’exploration

---

## ♿ Accessibilité à anticiper

Dès la structure HTML, il faut penser à :

- des titres clairs
- une hiérarchie logique
- des textes compréhensibles
- des labels clairs pour le formulaire
- des contenus visuels accompagnés de texte utile

---

## 🧼 Résultat attendu

Avant même le CSS, le site doit déjà être :

- compréhensible
- bien structuré
- logique à lire
- propre dans son organisation

---

## ✅ Validation de l’étape

L’étape est validée si :

- la structure globale de chaque page est définie
- les zones sémantiques sont identifiées
- la hiérarchie des titres est pensée
- la navigation est prévue
- l’accessibilité est anticipée
